## A2A 协议学习笔记

### 核心概念
- **Agent Card**: JSON 元数据，描述智能体的身份、能力、服务端点、认证要求
- **Task**: 工作单元，有状态生命周期
- **Message**: 通信回合，包含 role (user/agent) 和多个 Part
- **Part**: 最小内容单元，可以是文本、文件引用或结构化数据

### 协议特点
- 基于 JSON-RPC 2.0 over HTTP(S)
- 支持同步、流式、异步推送
- 支持长时任务和人在回路场景
- 智能体可以协作而不暴露内部状态

### 核心操作
1. `Send Message` - 发送消息
2. `Send Streaming Message` - 流式发送
3. `Get Task` - 获取任务状态
4. `List Tasks` - 列出任务
5. `Cancel Task` - 取消任务
6. `Subscribe to Task` - 订阅任务更新

### SDK
- JS SDK: `npm install @a2a-js/sdk`
- Python SDK: `pip install a2a-sdk`

## 实现思路

要让若兰和阿轩通过 A2A 通信，需要：

### 1. 作为 A2A Server (被调用方)
- 暴露一个 HTTP 端点
- 提供 Agent Card（描述我的能力）
- 处理 incoming message，转发给 OpenClaw

### 2. 作为 A2A Client (调用方)
- 获取阿轩的 Agent Card
- 调用 Send Message 发送消息
- 接收阿轩的回复

### 3. 可能的实现方式
- 创建一个 A2A skill，包含 server 和 client 功能
- Server: 一个 Express/Fastify 服务，监听 A2A 请求
- Client: 使用 @a2a-js/sdk 调用其他智能体

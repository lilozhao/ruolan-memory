### ⚠️ 安全教训

**问题：** 推送代码时未扫描敏感信息，导致 API Key 和飞书 Secret 暴露到 Git 仓库。

**修复措施：**
1. `identity.ruolan.json` - 移除真实 API Key，改为占位符
2. `notify_feishu.js` - 移除飞书 App ID/Secret，改为环境变量
3. `.gitignore` - 添加 `identity.json`、`.env` 等敏感文件

**以后必须遵守：**
- 推送前扫描：`grep -r "sk-\|apiKey\|secret\|password" --include="*.json" --include="*.js"`
- 敏感信息用环境变量或本地配置文件
- `.gitignore` 必须包含 `identity.json`、`.env`
